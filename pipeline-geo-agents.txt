Archivo: crates/geonexus-tauri/src/commands/chat/mod.rs

Archivo: crates/geonexus-tauri/src/commands/chat/send_message.rs
Lineas: 38-598 *** ARCHIVO PRINCIPAL DEL PIPELINE ***
Proposito: Funcion send_message() - corazon del pipeline. Pasos:
  1. validate() input
  2. ensure_conversation() - crea o reusa conversacion
  3. Inserta mensaje user en DB
  4. classify_intent() - clasifica por keywords
  5. Consulta graph_nodes + graph_edges de DB
  6. build_graph_context() - filtra nodos por intent
  7. RAG: llama sidecar Python recall_chunks (ChromaDB)
  8. build_project_context via sidecar Python
  9. Construye mention context (@mentions)
  10. Web search via sidecar Python (si web_search=true)
  11. build_messages() - arma system prompt + contextos + historial
  12. LOOP tool-calling (max 10 iteraciones):
       - Llama sidecar Python chat_llm_stream
       - Si hay tool_calls: ejecuta y sigue
       - Si hay content: break
  13. validate_response() - verifica articulos/capas mencionados
  14. Guarda analysis_session (si no es ConsultaGeneral)
  15. extract_message_stats() - tokens, tiempo, velocidad
  16. Guarda mensaje assistant en DB
  17. extract_chat_entities via sidecar Python -> grafo
  18. Emite graph:updated event
  19. Retorna SendMessageResponse

Archivo: crates/geonexus-tauri/src/commands/chat/messages.rs
Lineas: 5-83 *** SYSTEM PROMPT ***
Proposito: build_messages() - construye el array de mensajes para el LLM:
  - System prompt: "Eres Geo Agents, un asistente experto..."
  - Project context (assets + nodos del grafo)
  - Web search context (resultados de busqueda)
  - RAG context (chunks de documentos)
  - Historial (ultimos 20 mensajes)
  - Mensaje del usuario

Archivo: crates/geonexus-tauri/src/commands/chat/classifier.rs

  - que capas/tengo/existe -> DescubrimientoDatos

Archivo: crates/geonexus-tauri/src/commands/chat/stats.rs

Archivo: crates/geonexus-tauri/src/commands/llm/sidecar.rs

 3. PYTHON SIDECAR
================================================================================

Archivo: ai/sidecar.py
Lineas: 12-163
Proposito: Entry point del sidecar. Dispatcher CLI que recibe --action:
  - recall_chunks -> query ChromaDB
  - build_project_context -> contexto del proyecto
  - chat_llm_stream -> streaming LLM
  - search_web -> busqueda web
  - extract_chat_entities -> extraer entidades al grafo
  Funcion _print(): json.dumps + sys.stdout.buffer.write (UTF-8).

Archivo: ai/llm/router.py
Lineas: 303-511
Proposito: chat_llm_provider_stream() - enrutador de LLM con streaming:
  - Ollama: POST {url}/api/chat, decode_unicode=True
  - OpenAI-compatible: POST {url}/chat/completions, SSE
  - Anthropic: POST {url}/v1/messages, SSE
  - OpenRouter: POST {url}/chat/completions con headers
  Streamea tokens como {"type":"delta","content":"..."}
  Finaliza con {"type":"done","status":"ok","message":...,"usage":...}

Archivo: ai/memory/chroma.py
Lineas: 1-158
Proposito: query_chunks() - consulta ChromaDB por similitud coseno.
           Retorna chunks con text, source, asset_id, score.

Archivo: ai/memory/embeddings.py
Lineas: 1-73
Proposito: Genera embeddings via OpenAI API, Ollama, o fallback
           deterministico (SHA-256 -> seeded RNG -> vector unitario).

Archivo: ai/context/builder.py
Lineas: 1-51
Proposito: build_project_context() - consulta SQLite directamente para
           obtener assets indexados y nodos del grafo del proyecto.

Archivo: ai/web_search.py
Lineas: 1-201
Proposito: Busqueda web via DuckDuckGo, Google, Bing, o SerpAPI.
           Retorna hasta 5 resultados con url, title, snippet.

Archivo: ai/graph/chat_extractor.py
Lineas: 58-180
Proposito: Extrae entidades de la conversacion (definiciones, acronimos,
           terminos, referencias a articulos/zonas) y las inserta en
           el grafo de conocimiento.

================================================================================
 4. FLUJO COMPLETO (RESUMEN)
================================================================================

Usuario escribe mensaje
  -> ChatComposer.tsx (textarea + @mentions)
    -> useChatSession.ts (submit)
      -> Crea mensaje optimista user + placeholder assistant
      -> Fases: classifying (0s) -> searching (0.8s) -> generating (3.5s)
      -> Escucha "llm:token" para streaming
      -> api/chat.ts sendMessage()
        -> invoke("send_message", { input })
          -> Rust: send_message.rs
            1. validate input
            2. ensure/create conversation
            3. Insert user message a SQLite
            4. classify_intent() por keywords
            5. Query graph nodes + edges de DB
            6. build_graph_context() filtrado por intent
            7. Python sidecar: recall_chunks (ChromaDB RAG)
            8. Python sidecar: build_project_context
            9. Build mention context de @mentions
            10. Python sidecar: search_web (si activado)
            11. build_messages() -> system prompt + 4 capas de contexto + historial
            12. LOOP tool-calling (max 10):
                  Python sidecar: chat_llm_stream (HTTP POST al LLM)
                    -> Streams tokens via stdout
                    -> Rust lee lineas, emite "llm:token" events
                    -> Frontend acumula tokens en mensaje assistant
                  Si tool_calls: execute_tool_call() -> append result -> loop
                  Si content: break
            13. validate_response()
            14. Save analysis_session (si no es general)
            15. extract_message_stats()
            16. Save assistant message a SQLite
            17. Python sidecar: extract_chat_entities -> grafo
            18. Emit graph:updated event
            19. Return SendMessageResponse
      -> Frontend recibe respuesta
        -> Actualiza mensaje assistant: content + stats + knowledgeSteps
        -> Fase: extracting -> idle
      -> ChatTranscript renderiza AssistantMessage
        -> ThinkingInline (pasos completados)
        -> MarkdownContent (ReactMarkdown)
        -> TokenStatsBadge (circular con tokens y tiempo)
        -> ActionSuggestions (sugerencias)

================================================================================
 5. SISTEMA DE CONTEXTO (4 CAPAS)
================================================================================

El LLM recibe estos mensajes system ANTES del historial y el mensaje user:

1. PROJECT CONTEXT (messages.rs:26-31)
   - Assets indexados del proyecto
   - Nodos del grafo de conocimiento
   Fuente: context/builder.py + send_message.rs:146-154

2. GRAPH CONTEXT (context.rs:82-126)
   - Nodos filtrados por tipo de consulta (intent)
   - Relaciones entre nodos
   Fuente: send_message.rs:77-105

3. @MENTION CONTEXT (send_message.rs:157-218)
   - "[CONTEXTO ADJUNTO: Conector/ Documento/ Nodo...]"
   Fuente: DB queries en send_message.rs

4. WEB SEARCH CONTEXT (messages.rs:33-41)
   - "[Fuente 1] Title - URL: snippet"
   Fuente: web_search.py via send_message.rs:229-266

5. RAG CONTEXT (messages.rs:43-48)
   - "[1] chunk text" desde ChromaDB
   Fuente: memory/chroma.py via send_message.rs:116-144
