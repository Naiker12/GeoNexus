3. PROBLEMAS CRITICOS (Por que no responde)
===================================================

  3.1 NO HAY LISTENER FRONTEND (BLOQUEANTE)
  ───────────────────────────────────────────
    - telegram.rs linea 200: app.emit("telegram:message", payload)
    - El evento se emite PERO NADIE lo escucha
    - Busqueda en todo el frontend: 0 resultados para "telegram:message"
    - No hay ningun listen("telegram:message", ...) en ningun archivo
    - RESULTADO: El bot recibe el mensaje, lo procesa, pero no hace
      nada con el. El usuario nunca recibe respuesta.

  3.2 NO HAY PIPELINE DE RESPUESTA (BLOQUEANTE)
  ───────────────────────────────────────────
    - Aunque existiera un listener, NO HAY logica para:
      a) Procesar la pregunta del usuario
      b) Llamar al LLM para generar respuesta
      c) Enviar la respuesta de vuelta a Telegram
    - La funcion sendTelegramMessage() EXISTE en la API
      (src/api/telegram.ts linea 61) pero NUNCA se llama
      en respuesta a un mensaje entrante

  3.3 NO HAY "ESCRIBIENDO..." / CHAT ACTION (BLOQUEANTE)
  ───────────────────────────────────────────
    - Telegram permite enviar sendChatAction para mostrar:
      "escribiendo...", "enviando foto...", "buscando..."
    - El backend NO implementa sendChatAction
    - El usuario envia un mensaje y NO ve ninguna indicacion
      de que el bot esta procesando
    - RESULTADO: El usuario piensa que el bot no funciona

  3.4 BUG: StopIcon NO IMPORTADO
  ───────────────────────────────────────────
    - TelegramIntegrationPanel.tsx linea 319:
        <StopIcon className="size-3.5" />
    - El import en linea 2-10 SOLO importa StopCircleIcon
    - StopIcon NO existe en lucide-react (es StopCircleIcon)
    - RESULTADO: Error de compilacion/TypeScript.
      El panel de configuracion puede no renderizarse
      o mostrar un error en la consola.

===================================================
4. PROBLEMAS SECUNDARIOS
===================================================

  4.1 handle_update() SOLO PROCESA TEXTO
  ───────────────────────────────────────────
    - telegram.rs linea 192: if let Some(text) = message.text
    - Fotos, documentos, voz, stickers, ubicaciones son IGNORADOS
    - No hay logging ni error cuando se ignora un mensaje no-texto

  4.2 Necesidad de sendMessage con parse_mode escapado
  ───────────────────────────────────────────
    - sender.rs linea 15 usa "MarkdownV2"
    - MarkdownV2 requiere "escapado" de caracteres especiales
      (_ * [ ] ( ) ~ > # + - = | { } . !)
    - Las respuestas del LLM pueden contener estos caracteres
    - Si no se escapan, Telegram devuelve error 400 "Bad request"
    - No hay funcion de escape en el codigo actual

  4.3 Sin verificar que el bot NO responda en grupos no autorizados
  ───────────────────────────────────────────
    - El chat puede ser un grupo
    - No hay validacion de que el bot solo responda en chats privados
      si esa es la intencion

  4.4 Polling infinito sin heartbeat/timeout exterior
  ───────────────────────────────────────────
    - start_polling_loop() ejecuta loop infinito
    - Si el token se revoca, solo imprime a stderr y sigue
    - No hay manera de detectar "token invalido" desde afuera
    - stop_polling usa abort() que es forzado

  4.5 Sin soporte de webhook
  ───────────────────────────────────────────
    - Solo usa long polling (getUpdates)
    - En produccion seria mejor webhook
    - No hay comando para registrar/setear webhook
