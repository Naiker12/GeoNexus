## 3. Problemas Pendientes / Áreas de Mejora

### 3.1. Refactorización de send_message.rs
**Archivo**: `crates/geonexus-tauri/src/commands/chat/send_message.rs`
**Problema**: Demasiado código en un solo archivo (~1000 líneas)
**Sugerencia**: Dividir en submódulos:
1. `crates/geonexus-tauri/src/commands/chat/send_message/context.rs`: Preparación de contexto (rag, project, web, etc.)
2. `crates/geonexus-tauri/src/commands/chat/send_message/tool_loop.rs`: Bucle de tool-calling
3. `crates/geonexus-tauri/src/commands/chat/send_message/streaming.rs`: Gestión del streaming del sidecar
4. `crates/geonexus-tauri/src/commands/chat/send_message/mod.rs`: Comando principal unificado

### 3.2. Tauri Build Script Panic
**Problema**: El build script de geonexus-tauri a veces panica en Windows
**Ubicación**: `crates/geonexus-tauri/build.rs`
**Nota**: Este problema no es causado por nuestros cambios, pero es un problema existente.

### 3.3. Frontend: Testing Coverage
**Observación**: Existen tests básicos (`src/__tests__/`), pero la cobertura puede ampliarse.

### 3.4. Análisis de Código Duplicado
**Sugerencia**: Utilizar herramientas como `cargo clippy` para Rust y `eslint` para TypeScript para encontrar duplicaciones y smells.

### 3.5. Archivos Temporales
**Observación**: send_message.rs crea archivos temporales para mensajes y tools JSON. Asegurarse de que siempre se borren (aunque hay código para borrar, puede haber edge cases).
