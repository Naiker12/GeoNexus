╔════════════════════════════════════════════════════════════════════════════╗
║   HOOKS PRINCIPALES (src/hooks/)                                        ║
╚════════════════════════════════════════════════════════════════════════════╝

  useChat.ts:
     ├── Gestiona el estado del chat (mensajes, conversación, loading)
     ├── Envía mensajes vía Tauri invoke("send_message")
     ├── Maneja streaming de respuesta
     ├── Eventos de razonamiento (reasoning:thinking, reasoning:tool)
     └── Integración con MCP tools + skills + context graph

  useReasoningTimeline.ts:
     ├── Escucha eventos de razonamiento del backend
     ├── reasoning:thinking → texto de pensamiento en vivo
     ├── reasoning:tool → herramientas utilizadas
     ├── reasoning:sub_item → ítems del plan
     ├── reasoning:complete → razonamiento finalizado
     └── reasoning:error → errores de razonamiento

  useMcpServers.ts:
     ├── Lista servidores MCP registrados
     ├── Ping, registro, eliminación
     └── Estado online/offline

  useMcpTools.ts:
     ├── Lista herramientas por servidor
     └── Estado de carga

  useMcpTerminal.ts:
     ├── Consola interactiva MCP
     └── Historial de comandos

╔════════════════════════════════════════════════════════════════════════════╗
║   SISTEMA DE EVENTOS (FRONTEND)                                         ║
╚════════════════════════════════════════════════════════════════════════════╝

  El frontend recibe eventos del backend a través de Tauri event system:
     - listen("event-name", callback)
     - Eventos emitidos desde Rust con app_handle.emit()

  Eventos clave:
     reasoning:thinking  → Texto de razonamiento del LLM (streaming)
     reasoning:tool      → Herramienta siendo ejecutada
     reasoning:sub_item  → Sub-ítem de plan (file changes, etc.)
     reasoning:complete  → Razonamiento finalizado
     reasoning:error     → Error en razonamiento
     agent:*             → Eventos del agente de código (agent mode)
     mcp:*               → Eventos de MCP

  Diferencia entre modos:
     - Chat mode: usa reasoning:* events + AssistantMessage + useReasoningTimeline
     - Agent mode: usa agent:* events + CodingAgentPanel
