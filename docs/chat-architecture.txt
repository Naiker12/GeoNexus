RUTA: src/components/chat/useChatSession.ts  (338 lines)

STATE INTERNO:
  messages: Message[]           = []
  conversationId: string|null   = loadConversationId()  (localStorage)
  pending: boolean              = false
  error: string|null            = null
  loadingHistory: boolean       = false
  contextToggles: ContextToggle = { rag_chunks:true, indexed_assets:true,
                                    graph_nodes:true }
  webSearchEnabled: boolean     = loadWebSearchEnabled() (localStorage)

PERSISTENCIA:
  localStorage("geonexus.activeConversationId") — ID de conversación activa
  localStorage("geonexus.webSearchEnabled")     — Estado de búsqueda web

VARIABLES MÓDULO:
  let researchTimerId — fuera del hook (ámbito módulo), usada x el timer de 500ms
                        → se limpia en finally y en newConversation

FUNCIÓN INTERNA:
  updateAssistantMessage(id, updates) — merge parcial de updates en un assistant
    message existente por id. Usado para actualizar el placeholder de búsqueda
    cuando llega la respuesta.

COMPORTAMIENTO DE submit(content):
  1. Validar: !pending, content.trim(), activeProvider
     ─ Si no hay activeProvider: toast "Sin proveedor" y return
  2. Optimistic user message:
     id="local-{Date.now()}", role="user", content=clean
  3. Construir SendMessageInput con contextToggles y webSearch
  4. Si webSearchEnabled:
     ─ Crear placeholder assistant:
       id="research-{Date.now()}", isSearching=true, researchSources=[]
     ─ Iniciar timer cada 500ms que actualiza:
       searchElapsedSeconds, currentSearchQuery
         <2s:  query original
         <4s:  query + "análisis de resultados"
         >=4s: query + "generando respuesta"
  5. await sendMessage(input)  →  invoke("send_message", { input })
  6. Éxito:
     ─ Si webSearch: updateAssistantMessage(placeholderId, {
         isSearching:false, researchSources:response.research_sources,
         content:response.message.content, stats:response.message.stats, ...})
     ─ Si no webSearch: agregar response.message a messages[]
     ─ Toast "GeoNexus ha completado el análisis" (éxito)
  7. Error:
     ─ Filtrar mensajes optimistas
     ─ Mostrar toast error
  8. finally: setPending(false), clearInterval(researchTimerId)

COMPORTAMIENTO DE regenerate():
  1. Buscar el último user message en messages[]
  2. setMessages: eliminar desde ese user message hasta el final
  3. submit(lastUserContent) — re-envía el mensaje

COMPORTAMIENTO DE newConversation():
  ─ setMessages([]), setConversationId(null), clearInterval(researchTimerId)
  ─ Limpia localStorage activeConversationId

RETURN:
  { activeProvider, conversationId, error, messages, pending,
    loadingHistory, contextToggles, setContextToggles,
    webSearchEnabled, setWebSearchEnabled, submit, regenerate,
    loadConversation, newConversation }

  #[tauri::command] send_message(state, input) → SendMessageResponse

  FLUJO COMPLETO:
    1. Validar input (input.validate())
    2. Generar trace_id (UUID v4)
    3. Asegurar conversación (ensure_conversation)
    4. Crear user Message, insertar en DB (chat_repo::insert_message)
       ─ Si nueva conversación: generar título (title_from_message)
    5. RAG context:
       ─ run_sidecar_json::<Vec<RecallChunk>>(["--action","recall_chunks",
           "--query",input.content,"--project_id",input.project_id,"--top_k","4"])
       ─ Formatear como contexto textual "[1] texto..."
    6. Project context:
       ─ run_sidecar_json::<Value>(["--action","build_project_context",
           "--project_id",input.project_id])
       ─ Extraer "context" string
    7. Web search (si input.web_search):
       ─ extract_search_query(input.content) — limpia query
       ─ run_sidecar_json::<Vec<ResearchSource>>(["--action","search_web",
           "--query",search_query,"--max_results","5"])
       ─ Formatear como contexto "[Fuente 1] title - url: snippet"
    8. Tool definitions:
       ─ get_tool_definitions() → 4 tools (read_file, search_code,
         list_directory, glob_files)
    9. Build messages array:
       ─ chat_repo::list_messages (historial)
       ─ context::build_messages(history, project_context, web_context,
         rag_context, user_content)
       ─ Sistema: "Eres GeoNexus IA..." + contextos + historial + msg actual
   10. Tool-calling loop (máx 10 iteraciones):
       ─ run_sidecar(["--action","chat_llm","--provider_type",...,
           "--messages",messages_json,"--tools",tools_json])
       ─ Parsear SidecarChatResult (incluye usage: stats de tokens)
       ─ Si tiene tool_calls:
           push assistant(tool_calls) + tool(result)
           execute_tool_call(tc)
           continue
       ─ Si no: extraer final_content, break
   11. Extraer MessageStats del sidecar response:
       ─ extract_message_stats(SidecarChatResult) → Option<MessageStats>
       ─ Campos: input_tokens, output_tokens, total_tokens, duration_ms,
         tokens_per_second, cost_usd, context_window, context_used_pct
   12. Guardar assistant Message:
       ─ sources = recall_chunks[].source
       ─ research_sources = si hay resultados de búsqueda
       ─ stats = extracted_stats
       ─ chat_repo::insert_message
   13. Return SendMessageResponse {
         conversation_id, message, chunks_used: vec![], trace_id,
         research_sources, search_query
       }

    extract_search_query() — limpia query:

      ─ Trunca a 80 chars
      ─ Capitaliza primera letra
    extract_message_stats() — NO documentado antes:

      ─ Calcula duration_ms si hay timestamp
      ─ Calcula tokens_per_second = total/duration
      ─ Calcula cost_usd según modelo aprox.
      ─ Obtiene context_window de model_context_window()
      ─ Calcula context_used_pct

6a. Sidecar Execution
────────────────────────────────────────────────────────────────────────────────
  run_sidecar(args: &[&str]) → Result<String, String>
    ─ Llama run_sidecar_with_env(args, None)

  run_sidecar_with_env(args, env_var) → Result<String, String>:
    1. project_root(): buscar ai/sidecar.py subiendo directorios
    2. python_executable(): buscar en orden:
       ai/.venv/Scripts/python.exe  (Windows, alta prioridad)
       .venv/Scripts/python.exe     (Windows)
       ai/.venv/bin/python          (Unix)
       .venv/bin/python             (Unix)
       "python"                     (fallback sistema)
    3. Ejecutar: python ai/sidecar.py [args], PYTHONIOENCODING=utf-8
    4. Capturar stdout
    5. Si status.success: Ok(stdout)
       Si no: Err("Error en sidecar Python: {stderr} {stdout}")

────────────────────────────────────────────────────────────────────────────────

PROBLEMA:  Los botones de copiar, editar y regenerar estaban siempre visibles
           debajo de cada mensaje. El usuario reportó que deberían aparecer
           solo al hacer hover (pasar el mouse), como es estándar en UIs de chat.

CAMBIO APLICADO (versión final solicitada por usuario):
  ─ Los iconos están siempre visibles (sin hover para mostrar/ocultar)
  ─ Están FUERA de la burbuja, debajo con pt-0.5 de separación mínima
  ─ No hay hover que los oculte — siempre a la vista
  ─ MessageActions.tsx:
      • CopyButton: sin pill contenedor, solo <button> directo con Tooltip
      • UserActions: envuelto en <>, sin pill, botones directos
      • Botones: p-1 rounded-md, hover:bg-muted/60 (solo hover de feedback)
  ─ ChatTranscript.tsx (user messages): botones debajo del bubble, siempre vis
  ─ AssistantMessage.tsx: botones debajo del contenido, siempre visibles
