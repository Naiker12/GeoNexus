
# GeoNexus - Audit: Skills, MCP, Chat Systems
## Date: 2026-06-21

## 1. Skills System Overview
### Backend (Rust)
#### Files
- `crates/geonexus-tauri/src/commands/skills.rs`:
  - `list_skills`: Retrieve all installed skills
  - `install_skill_from_file`: Install from local MD file
  - `toggle_skill`: Enable/disable a skill
  - `read_skill_md`: Read a skill's markdown content
  - `preview_skill_file`: Preview local skill files
  - `install_skill_from_github`: Install from a GitHub URL (converts to raw content)

- `crates/geonexus-db/src/skills/parser.rs`:
  - `parse_skill_md`: Extract frontmatter YAML and body from skill files
  - `compute_hash`: SHA256 hash of skill content

- `crates/geonexus-db/src/skills/injector.rs`:
  - `select_skills_for_goal`: Simple keyword-based skill relevance scoring for a user's goal
  - `build_skills_context`: Format skills as a section for system prompt injection
  - Score calculation: 1pt per description word match, 2pts per exact tag match

- `crates/geonexus-db/src/skills/registry.rs`:
  - Full CRUD operations for skills in DB
  - Activation tracking per skill/conversation

### Frontend (TypeScript/React)
- `src/api/skills.ts`: API hooks for skills
- `src/features/workspace/skills/`: Full UI:
  - List/grid view for skills
  - Install dialog
  - Detail drawer

### Integration with Chat Flow
- In `send_message.rs` (chat command):
  - Skills from `input.skill_names` are explicitly read and injected
  - Skills are added to the system prompt as a "## Skills activos" section
  - Skill activations are recorded in DB

## 3. Chat System Overview
### Flow (from send_message.rs)
1. Input Validation (`validate()`)
2. Intent Classification (`classify_intent()`)
3. Graph Context Retrieval (relevant graph nodes & edges)
4. RAG Recall (vector DB chunks)
5. Web Search (if enabled)
6. Tool Catalog Loading (local + MCP tools)
7. Skills Context Injection
8. Message History Building
9. Tool Calling Loop (max 10 iterations)
10. Response Validation
11. Episodic Memory Saving (analysis sessions)
12. Final Message Saving

### Event Systems
1. **Legacy Reasoning Timeline**:
   - Events: `reasoning:start`, `reasoning:step`, `reasoning:sub_item`, `reasoning:end`
   - Used in `useReasoningTimeline.ts`
   
2. **New AgentTraceEvent System** (just implemented):
   - Hierarchical events with parent IDs
   - Persisted in `messages.reasoning_events` DB column
   - Rendered by new `TraceTreeView.tsx`

## 4. Key Findings & Improvement Opportunities

### Skills System
- **Current Issue**: Relevance scoring is very basic (keyword matching only)
  - Could be improved with simple embeddings-based similarity
- **Opportunity**: Skill versioning & auto-updates from GitHub

### MCP System
- **Code Duplication**: STDIO tool discovery logic duplicated in `register_server`, `ping_mcp_server`, `discover_mcp_tools`
- **Opportunity**: Better error categorization & retry logic
- **Opportunity**: MCP resource support (not just tools)

### Chat System
- **Duplicate Logic**: Project context retrieval partially duplicated
- **Opportunity**: Streamlined context building with clear priority ordering
- **Opportunity**: Token budget estimation for context trimming
