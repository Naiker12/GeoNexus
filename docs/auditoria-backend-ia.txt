El backend IA tiene DOS modos de operación:

A) CLI subprocess (original) — `sidecar.py --action <accion> [args]`
   Tauri invoca como subproceso via std::process::Command. Un solo
   request/response por invocación.

B) WebSocket Gateway persistente — `sidecar.py --serve`
   Levanta un servidor FastAPI/Uvicorn en ws://127.0.0.1:9876/ws.
   Tauri (y terminal_client.py) se conectan via WS y envían acciones
   como mensajes JSON. El gateway mantiene sesiones PersistentShell
   entre requests.

Ruta base: D:\GeoNexus\ai\
Intérprete: .venv\Scripts\python.exe (virtual env)
Entry points: config.py → server.py (modo --serve)

Características del router:
- Tool calling (function calling) en chat sincrónico y streaming
- Tool calls se agregan incrementalmente durante streaming
- Reasoning content (thinking) expuesto via streaming
- Rate limiting para proveedores locales
- Fallback a determinista si no hay API key
- Normalización de usage entre proveedores

Variables de entorno requeridas según proveedor:
- OPENAI_API_KEY, ANTHROPIC_API_KEY, OPENROUTER_API_KEY

6. MEMORIA VECTORIAL (memory/)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

chroma.py:
- Cliente ChromaDB real con fallback a MockChromaClient local (JSON)
- Mock almacena embeddings en archivos JSON bajo ./chroma_db/
- Operaciones: upsert, query con filtro where
- Cálculo de similitud: coseno

embeddings.py:
- Tres estrategias en orden de prioridad:
  1. OpenAI embeddings (text-embedding-3-*)
  2. Ollama embeddings (http://localhost:11434/api/embeddings)
  3. Determinista (SHA-256 → seed → RNG gauss → vector unitario)
- Dimensión por defecto: 1536

fs_embed.py (CLI independiente):
- Embedding determinista local (SHA-256 como seed)
- Cache en .fs_embed_cache/ (archivos JSON por archivo)
- Comandos: ping, index, search
- Dimensión: 384

7. PIPELINE DE INDEXACIÓN (pipeline/indexer.py)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Flujo completo:
1. extract_text(file_path) → texto plano (reader.py: TXT, PDF, DOCX)
2. chunk_text(text) → lista de chunks (chunker.py: 800 palabras, overlap 150)
3. generate_embeddings(chunks) → vectores (embeddings.py)
4. upsert_chunks_to_vector_store(ids, embeddings, docs, metadatas) → ChromaDB
5. extract_graph_entities(chunks) → nodos + aristas (graph/extractor.py)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

extractor.py (desde documentos):
- Detecta: Artículos (Art. N), Zonas (Zona/Sector Nombre),
  conceptos predefinidos (20 conceptos GIS colombianos)
- Aristas: "aplica en", "define", "afecta", "asociado con"

chat_extractor.py (desde chat):
- Detecta: patrones de definición, acrónimos (3-6 mayus),
  términos multi-word, tecnologías comunes
- Stop phrases y stop definitions

layout.py:
- Posicionamiento radial por clusters semánticos
- Orden: concepto → norma → zona → capa → documento → chat_turn →
  web_search → upload → connector → rag_recall

memory.py:
- Algoritmo Ebbinghaus: score = base_weight * e^(-t/stability) + frequency_bonus
- Nodos pinned: score mínimo 5.0

16. INTEGRACIÓN CON TAURI (Rust → Python)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DOS MODOS DE COMUNICACIÓN:

A) Gateway WebSocket (preferido):
   - Tauri lanza sidecar.py --serve al iniciar la app
   - GatewayClient (Rust) se conecta a ws://127.0.0.1:9876/ws
   - Envía acción como JSON: {action, params, session_id}
   - Stateless: cada llamada abre conexión fresca
   - Fallback: si gateway no disponible, pasa a modo subprocess

B) Subprocess CLI (fallback):
   - Tauri invoca sidecar.py como subproceso
   - Pasa argumentos CLI: --action <accion> --param1 valor1
   - Lee JSON de stdout (una línea = una respuesta)

STREAMING:
- El gateway envía eventos delta/thinking/done uno por uno via WS
- GatewayClient::send_stream() retorna un GatewayStream (impl Stream)
- run_sidecar_streaming() procesa eventos y emite eventos Tauri al frontend
- Eventos: llm:token, reasoning:thinking, chat:preview_chunk

COMANDOS TAURI RELACIONADOS:
- run_health_check → incluye gateway_connected
- check_gateway → bool (health check rápido)
- send_message → usa run_sidecar_streaming (gateway first)
