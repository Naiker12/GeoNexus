FLUJO DE EJECUCIÓN MCP:
1. Frontend llama a invoke("call_mcp_tool", {server_id, tool, args})
2. Tauri handler → registry::get_server() → executor::call_tool_for_server()
3. Executor verifica allowlist + rate limit
4. Según transporte: HTTP (handshake + POST) o STDIO (spawn + JSON-RPC)
5. Sanitiza respuesta (elimina secrets)
6. Audita (inserta en mcp_tool_calls)
7. Retorna CallToolResult al frontend

7. CRYPTO (crypto.rs)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Algoritmo: AES-256-GCM
Derivación de key:
1. Si GEONEXUS_SECRET_KEY existe → usa esa key (base64, 32 bytes)
2. Sino → SHA-256("geonexus-telegram-v1:" + machine_id)
   - Windows: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Cryptography\MachineGuid
   - Linux: /etc/machine-id o /var/lib/dbus/machine-id
   - Fallback: "geonexus-fallback-id"
Formato: nonce_base64:ciphertext_base64
Fingerprint: SHA-256(token)[:8] en hex

Tests incluidos: encrypt/decrypt roundtrip, tamper detection, fingerprint consistency

Implementado en tauri::main.rs (inline):

- Poll interval: 2000ms
- Handlers: FilesystemWorker + default_handlers (de geonexus-core::workers::tasks)
- Ejecuta en tokio runtime propio (spawn en std::thread)
- Toma tareas de agent_tasks (pop_pending transaccional)

NOTA: geonexus-core::workers NO EXISTE como archivos. Es código
planificado. El WorkerPool y default_handlers están referenciados
pero no implementados.
