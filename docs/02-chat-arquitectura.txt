    ├── regenerate() → reenvía último mensaje
    ├── loadConversation(id) → carga historial
    └── newConversation() → limpia estado

    Submit flow:
      1. Crea mensaje user optimista
      2. Crea placeholder assistant
      3. Escucha evento "llm:token" para streaming
      4. Llama sendMessage(input)
      5. Actualiza mensaje assistant con respuesta real
      6. Maneja errores (rollback optimista)

3. RUST PIPELINE (send_message.rs — 658 líneas)
════════════════════════════════════════════════════════════════════════════

  Orden de ejecución:
    1. validate() input
    2. Crear/actualizar conversación + UUID trace_id
    3. Insertar user_msg en DB
    4. classify_intent(content) → QueryIntent
    5. Graph context: SELECT graph_nodes + edges
    6. RAG recall: Python sidecar recall_chunks (top_k=4)
       → Construye ChunkReference[] (asset_name, text_preview, score)
    7. Project context: sidecar build_project_context
    8. Mention context: lookup connector_configs, assets, graph_nodes
    9. Skills context: read_skill_md() por cada skill_name activo
       → "## Skills activos\n\n{contenido1}\n\n---\n\n{contenido2}"
   10. Web search: sidecar search_web (si web_search=true)
   11. build_messages() → mensajes system + context + history + user
   12. Tool-calling loop (max 10 iter):
       - Sidecar chat_llm_stream (primera llamada)
       - Si hay tool_calls, ejecuta execute_tool_call()
       - read_file / search_code / list_directory / glob_files
       - Continúa loop hasta respuesta final
   13. extract_message_stats() del último sidecar
   14. Insertar assistant_msg en DB
   15. Retornar SendMessageResponse

5. INTEGRACIONES ESPECIALES
════════════════════════════════════════════════════════════════════════════

  A. Skills:
     ├── Skill names enviados desde frontend en skill_names[]
     ├── Rust lee read_skill_md() de cada skill
     ├── Inyecta como system message "## Skills activos"
     ├── Skills vía @mention: se agregan a activeSkills automáticamente
     └── Skills desde SkillsPage "Usar en chat": dispatch custom event

  B. Agent Pipeline:
     ├── Tauri event "agent:event" con payload { agent, status, message, timestamp, data? }
     ├── ChatPanel muestra ReasoningEventStream durante ejecución
     └── 6 fases: Planner → Discovery → Knowledge → MCP → Reasoning → Result

  C. ChromaDB RAG:
     ├── Python sidecar _recall retorna chunks con chunk_index + chunk_id
     ├── Rust construye ChunkReference[] con lookup de asset_name
     ├── Frontend renderiza CitationsBlock colapsable
     └── Knowledge steps: chromadb → graph → assets

  D. Web Search:
     ├── Python sidecar search_web con max_results=5
     ├── DeepResearchPanel muestra fuentes en tiempo real
     └── research_timer actualiza elapsed seconds

  E. Streaming:
     ├── Evento Tauri "llm:token" para streaming de tokens
     ├── ChatPanel escucha y actualiza mensaje assistant en tiempo real
     └── loadingPhase: classifying → searching → generating → extracting
