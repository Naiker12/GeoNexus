╔════════════════════════════════════════════════════════════════════════════╗
║   4. PROBLEMA DE TOOL DISCOVERY — ANÁLISIS Y FIXES                      ║
╚════════════════════════════════════════════════════════════════════════════╝

  ┌─────────────────────────────────────────────────────────────────────────┐
  │ PROBLEMA: Las tools de servidores MCP STDIO no se descubrían           │
  │ automáticamente al registrarlos. El usuario debía hacer clic manual    │
  │ en "Descubrir tools" en cada tarjeta.                                  │
  └─────────────────────────────────────────────────────────────────────────┘

  Causa raíz #1 — Falta de auto-discovery en registro:
     handlers.rs register_mcp_server(): Sólo INSERT en SQLite,
     no llamaba a stdio::discover_tools() para servidores STDIO.
     handlers.rs import_mcp_config(): Ídem, no descubría tools.

     Fix: Ambas funciones ahora llaman stdio::discover_tools()
     después de registrar el servidor, y upsertan las tools en DB.

  Causa raíz #2 — Reader frágil en stdio.rs:
     read_json_line() leía UNA línea y fallaba si el servidor
     enviaba logs/debug antes del JSON, o si la respuesta tenía
     whitespace extra, o si el JSON llegaba fragmentado.

     Fix: read_json_line() ahora:
     • Itera hasta timeout progresivo
     • Salta líneas vacías
     • Salta líneas que no empiezan con { o [
     • Hace trim automático
     • Tolera respuestas multi-línea

  Causa raíz #3 — Protocol version incorrecto:
     stdio.rs usaba "protocolVersion": "2025-06-18" (inexistente).
     Algunos servidores MCP rechazaban o ignoraban la negociación.

     Fix: Cambiado a "2025-03-26" (versión oficial MCP).

  Causa raíz #4 — onDiscoverTools no estaba conectado:
     McpRouterSection.tsx renderizaba McpServerCard sin la prop
     onDiscoverTools. El botón "Descubrir tools" llamaba a
     undefined(), causando TypeError silencioso.

     Fix: Se agregó onDiscoverTools={async () => { await discoverStdioTools(server.id) }}
     y el import de discoverStdioTools.
