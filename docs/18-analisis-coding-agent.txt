========================================
1.  INTRODUCCIÓN
========================================
El Coding Agent permite al usuario escribir prompts en lenguaje
natural (ej: "crea un html con mi nombre Naiker") y que el sistema
genere automáticamente archivos de proyecto, los visualice en
tiempo real, y permita previsualizarlos.

Flujo de alto nivel:
  Usuario escribe prompt
    → ChatPanel.onSubmit()
    → agentState.mode === "agent"?
    → startGeneration(prompt)
    → Backend Rust (telegram.rs)
    → Eventos Tauri en tiempo real
    → useCodingAgentEvents() escucha 13 eventos
    → CodingAgentContext actualiza estado
    → CodingAgentPanel renderiza UI

========================================
3.  EVENTOS TAURI (13 LISTENERS)
========================================
Evento                   Payload                  Acción en reducer
───────────────────────  ───────────────────────  ─────────────────
agent:plan               { label, detail? }       SET_PLAN + status=planning
agent:step_start         { id, label, detail? }   ADD_EVENT + status=coding
agent:step_complete      { id, duration? }        UPDATE_EVENT → done
agent:step_error         { id, detail? }          UPDATE_EVENT → error
agent:file_created       { path, name, type,      ADD_FILE + ADD_EVENT
                           content?, language? }
agent:file_modified      { path, name }           UPDATE_FILE + ADD_EVENT
agent:tool_call          { id, tool, args? }      ADD_EVENT (running)
agent:tool_result        { id, result? }          UPDATE_EVENT (done)
agent:thinking           { text }                 ADD_EVENT (thinking)
agent:error              { message }              SET_ERROR + error
agent:preview_ready      { url }                  SET_PREVIEW_URL + ADD_EVENT
agent:cleanup_result     CleanupReport            SET_CLEANUP_REPORT
agent:done               void                     SET_STATUS done

========================================
4.  ESTADOS DEL AGENTE
========================================
  idle → thinking → planning → coding → done
                           ↘ error ↗

========================================
5.  DETECCIÓN DE INTENCIÓN (BACKEND)
========================================
El backend Rust analiza la description del prompt:

  Keywords                    Archivo generado
  ──────────────────────────  ────────────────
  "html", "pagina", "sitio"   index.html
  o empieza con "crea"/
  "haz"/"genera"
  "python", ".py", "script"   main.py
  "rust", ".rs"               main.rs
  default                     index.html

6.  CAMBIOS REALIZADOS (SESIÓN)
========================================
(a) Bug prompt vs description
    — Backend espera campo "description", no "prompt"

(b) Bug projectPath faltante
    — Se añadió projectPath: "./agent-projects/default"

(c) Error del agente no visible al usuario
    — Se muestra agentState.error en ChatComposer

(d) Eventos no coincidían con backend
    — Se alinearon nombres y payloads de eventos Tauri

(e) Scroll en timeline no funcionaba
    — Se añadió min-h-0 en la cadena flex

(f) Badge AGENTE no era clickeable
    — Se agregó toggle del panel con agentPanelOpen

(g) Paso actual no visible en barra superior
    — Se muestra el último evento en tiempo real

(h) FileTree sin orden ni visor
    — Directorios primero, split panel con <pre>/<code>
    — Visor de contenido al hacer click

(i) Backend hardcoded (siempre React/MapView)
    — Ahora usa description para decidir qué generar

(j) Código duplicado y archivos huérfanos
    — Se eliminaron CodingAgentCodeViewer.tsx,
      CodingAgentFileExplorer.tsx, CodingAgentPlanCard.tsx,
      CodingAgentPreview.tsx, CodingAgentTimeline.tsx
