  1. LAYOUT PRINCIPAL — Scroll solo en mensajes
================================================================================

ARCHIVOS MODIFICADOS:
  - src/index.css
  - src/App.tsx
  - src/features/workspace/GeoNexusWorkspace.tsx
  - src/components/chat/ChatPanel.tsx

PROBLEMA ORIGINAL:
  El scroll afectaba toda la pagina en vez del area de mensajes. El header
  se perdia y el footer se movia con los mensajes.

CAUSA RAICES DETECTADAS:
  a) #root usaba height: 100% sin overflow:hidden → el body scrolleaba
  b) Div interno del ChatPanel (flex flex-1 flex-col) sin min-h-0 →
     min-height:auto crecia al contenido, ignorando el flex-1 asignado
  c) Section del ChatPanel usaba flex-1 en vez de altura explicita

SOLUCION FINAL (3 cambios clave):

  1.1 index.css — Root con altura absoluta
      html, body, #root { height: 100vh; overflow: hidden; }
      → NUNCA hay scroll en la pagina. Todo overflow se maneja interno.

  1.2 ChatPanel — Altura explicita como las demas paginas
      section: h-[calc(100svh-3.5rem)]  ← mismo patron que DocumentsPage,
      → Altura DEFINIDA via calc(), no depende de cadena flex.               DataPage, AiContainersPage
      div interno: min-h-0 añadido ← permite respetar la altura de section

  1.3 Cadena completa de alturas:
      #root (100vh, overflow:hidden)
        SidebarProvider (min-h-svh flex row)
          Sidebar (h-svh) → row = 100svh
          SidebarInset (flex-col overflow-hidden)
            GeoNexusWorkspace (flex-1 min-h-0 overflow-hidden)
              AppTopbar (h-14 shrink-0) ← FIJO 56px, siempre visible
              main (flex-1 overflow-hidden flex-col)
                ChatPanel section (h-[calc(100svh-3.5rem)] flex-col overflow-hidden)
                  div (flex-1 min-h-0 flex-col) ← AHORA CON min-h-0
                    top bar (shrink-0) ← FIJO
                    messages (flex-1 overflow-y-auto min-h-0) ← UNICO SCROLL
                    composer (shrink-0 border-t) ← FIJO ABAJO

  3. ACTION SUGGESTIONS — Sugerencias clicables del modelo
================================================================================

ARCHIVOS NUEVOS:
  - src/utils/parseSuggestions.ts
  - src/components/chat/ActionSuggestions.tsx

ARCHIVOS MODIFICADOS:
  - src/components/chat/AssistantMessage.tsx
  - src/components/chat/ChatTranscript.tsx
  - src/components/chat/ChatPanel.tsx

QUE HACE:
  Cuando el modelo termina su respuesta con frases como:
    "Puedo ayudarte con:"
    "- item 1"
    "- item 2"
    "- item 3"

  parseSuggestions.ts detecta la frase gatillo al FINAL del contenido y
  extrae los items como sugerencias. El componente ActionSuggestions las
  renderiza como "pills" (rounded-full, border, bg-background, hover:bg-muted).

  Al hacer clic en una pill, se envia ese texto como mensaje de usuario
  (submit). El flujo completo:

    ChatPanel.submit()
      → ChatTranscript.onSendMessage
        → AssistantMessage.onSendMessage
          → ActionSuggestions.onSelect
            → submit(texto)

  Solo se parsea cuando NO esta en streaming (isStreaming=false).

FRASES GATILLO DETECTADAS (regex):
  - puedo ayudarte con:
  - te interesa?
  - en que puedo apoyarte?
  - por ejemplo:
  - aqui tienes (algunas/unas) sugerencias:
  - puedes preguntarme:

REQUISITOS:
  - Minimo 2 items de sugerencia
  - Items < 100 caracteres
  - Items con prefijo -, *, bullet point limpio

================================================================================
  NOTAS TECNICAS
================================================================================

  • Todas las otras paginas del workspace (DocumentsPage, DataPage,
    AiContainersPage) usan h-[calc(100svh-3.5rem)] overflow-auto para su
    layout y funcionan correctamente.

  • La diferencia clave entre esas paginas y ChatPanel es que ellas
    scrollean TODO el contenido de la pagina, mientras ChatPanel necesita
    scroll SOLO en el area de mensajes (header y footer fijos).

  • El div interno del ChatPanel necesitaba min-h-0 explicitamente porque
    en una columna flex, min-height:auto hace que el item crezca a su
    contenido en vez de respetar el tamano asignado por flex.
