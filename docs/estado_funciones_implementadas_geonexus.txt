GeoNexus - Estado de funciones implementadas vs plan backend Datos
Fecha de revision: 2026-06-08
Archivo base analizado: docs/plan_backend_datos_geonexus.txt

Resumen corto

GeoNexus tiene 9 fases planificadas. Fases 1-4, 6-9 implementadas; Fase 5 pendiente.

Estado general:
- Fase 1, inventario y metadata local: implementada en beta.
- Fase 2, conector local y cache: implementada en beta (10 tests).
- Fase 3, indexacion documental y vectorial: implementada en beta.
- Fase 4, containers-mcp: implementada para proveedor local (8 tests).
- Fase 5, OAuth OneDrive: pendiente.
- Fase 6, proveedor LLM base: implementada en beta (11 tests).
- Fase 7, chat real con memoria: completa (Bloques A+B: tipos, migraciones, repositorio, send_message, historial).
- Fase 8, auto-detect de modelos LLM: completa (list_llm_models nativo Rust, ProviderSetupDialog con badges y debounce).
- Fase 9, RAG recall + fuentes + contexto GIS: completa (Bloques C+D: recall semantico, sources, ProjectContextPanel).

Conclusion

GeoNexus ya tiene una columna vertebral backend bastante avanzada para Datos:
inventario, SQLite, conector local, eventos, indexacion, chunks, embeddings,
grafo y containers-mcp local. El chat real con RAG, contexto GIS, fuentes e

y las pruebas E2E.
