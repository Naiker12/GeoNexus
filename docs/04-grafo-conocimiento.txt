════ 1. ARQUITECTURA MODERNA ════════════════════════════════════════════════════

  Stack tecnológico completo:
    ├─ Base de datos: SQLite (2 tablas: graph_nodes, graph_edges)
    ├─ Backend: Rust + Tauri (11 comandos, CRUD + queries + eventos)
    ├─ Frontend: React/TypeScript + Canvas 2D + d3-force (simulación física)
    ├─ Extractores: Python (3 módulos: extractor, chat_extractor, layout)
    ├─ Integración: Chat + Indexación + Búsqueda web

  Principios de diseño:
    - Grafo vivo (actualizaciones en tiempo real)
    - Simulación física con d3-force (nodos se repelen, se atraen)
    - Canvas de alto rendimiento (no SVG, para miles de nodos)
    - Zoom y pan completos
    - Animaciones suaves y profesionales

════ 2. BASE DE DATOS ══════════════════════════════════════════════════════════

  Tabla graph_nodes: 23 campos
    ├─ Identidad: id, project_id, workspace_id
    ├─ Contenido: name, kind, description, evidence, icon
    ├─ Ubicación: x, y, weight (0-100)
    ├─ Trazabilidad: source_asset_id, source_chat_id, origin_kind
    ├─ Estado: pinned, is_ephemeral, deleted_at, extra
    ├─ Eventos: source_event, event_id
    ├─ Tiempo: created_at

  Tipos de nodos (10 kinds):
    - norma, documento, capa, zona, concepto
    - chat_turn, web_search, upload, connector, rag_recall

  Tabla graph_edges:
    - source, target, relation, strength (0-100)
    - Índices optimizados para consultas rápidas

════ 3. EXTRACTORES PYTHON ═════════════════════════════════════════════════════

  3a. extractor.py (159L) — Para documentos indexados
      - Extrae Art. N (regex) → kind: norma
      - Extrae Zona/Sector <Capitalizado> → kind: zona
      - 26 conceptos hardcodeados (retiro hídrico, zonificación, etc.)
      - Relaciones semánticas por co-ocurrencia

  3b. chat_extractor.py (180L) — Extracción desde el chat
      5 niveles de extracción:
        1. Definiciones ("X es un...", "se llama X")
        2. Acrónimos (3-6 mayúsculas)
        3. Términos multi-word capitalizados
        4. Tech keywords (Python, Rust, SQL, ML, etc.)
        5. Art./Zona (compatibilidad con extractor.py)

  3c. layout.py — Posicionamiento inicial por tipo

════ 4. FRONTEND: COMPONENTES MODERNOS ═════════════════════════════════════════
