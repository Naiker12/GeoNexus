═══ 1. ARQUITECTURA GENERAL ══════════════════════════════════════════════════

  SQLite (graph_nodes + graph_edges)
      ↓
  Rust: queries.rs + crud.rs (11 comandos Tauri registrados)
      ↓
  Frontend: GraphPage.tsx (641L) — SVG con d3-force
      ↓
  Python: extractor.py (documentos) + chat_extractor.py (chat)
      ↓
  DB: chat → LLM (injección de nodos en system prompt vía intent classifier)

  Stack completo: SQLite → Rust API → Tauri IPC → React/TypeScript + SVG.
  Sin Neo4j. Sin NLP semántico (solo regex en extractores).

═══ 2. BASE DE DATOS ═════════════════════════════════════════════════════════

  Tabla: graph_nodes (23 columnas tras migraciones)
    Columnas: id, project_id, workspace_id, name, kind, description,
    evidence, x, y, weight, created_at, source_asset_id (FK assets),
    source_chat_id (FK conversations), is_ephemeral, source_event,

    9 índices en nodes, 3 en edges.

═══ 3. EXTRACTORES PYTHON ═══════════════════════════════════════════════════

  3a. ai/graph/extractor.py (159L) — Para documentos indexados
      - Extrae Art. N (regex) → kind: "norma"
      - Extrae Zona/Sector <Capitalized> → kind: "zona"
      - 26 conceptos hardcodeados (retiro hidrico, zonificacion, etc.) → kind: "concepto"
      - Co-ocurrencia en chunks → aristas semánticas
      - Relaciones: "aplica en" (norma↔zona), "define" (norma↔concepto),
        "afecta" (concepto↔zona), "asociado con" (default)
      - Strength: 80
      - Fallback: nodo "documento" si no hay entidades
      - Sin NLP, sin deduplicación entre documentos
      - Llamado desde: commands/document/rebuild.rs vía sidecar --action extract_graph_entities

  3b. ai/graph/chat_extractor.py (180L) — Para extracción desde chat
      5 niveles de extracción en texto pregunta+respuesta:
        1. Patrones de definición ("X es un/una...", "se llama X") → weight=3
        2. Acrónimos (3-6 mayúsculas standalone, con stop-list) → weight=2
        3. Términos multi-word capitalizados (2-5 palabras) → weight=3
        4. Tech keywords (20: Python, Rust, SQL, ML, etc.) → weight 2-3
        5. Regex Art./Zona (compatibilidad con extractor.py) → weight 2-3
      - Stop-words: "hola", "gracias", etc. + palabras interrogativas
      - Aristas co-ocurrencia, strength=50
      - Llamado desde: send_message.rs línea 785 vía sidecar --action extract_chat_entities
      - Nodos creados con is_ephemeral=true

  3c. ai/graph/layout.py (59L) — Posicionamiento por tipo
      - Regiones semánticas (norma top-left, conceptos top-right, etc.)
      - Distribución uniforme + jitter ±2px dentro de cada región
      - reset_counters() para reiniciar contadores por tipo

  3d. ai/graph/graph.py (5L) — Placeholder (stub sin uso real)

═══ 4. RUST: COMANDOS TAURI (11 registrados) ═══════════════════════════════

  4a. commands/graph/queries.rs (154L) — Consultas
      - list_graph_nodes(project_id) → Vec<GraphNode>
      - list_graph_edges(project_id) → Vec<GraphEdge>
      - search_graph_nodes(project_id, query, kind?, limit?, offset?)
          → SearchGraphNodesResult { total, nodes }
          SQL: LIKE '%query%' en name, filtro kind opcional, paginación
          (default limit 20, max 100, offset 0), orden weight DESC
      - update_node_position(node_id, x, y) → persiste posición post-drag

  4b. commands/graph/crud.rs (255L) — Operaciones CRUD
      - delete_graph_node(id, force) → soft-delete (deleted_at), respeta pin
      - pin_node(id, pinned) → toggle 1/0
      - restore_graph_node(id) → deleted_at = NULL
      - list_orphan_nodes(project_id) → nodos sin source_asset_id ni source_chat_id
      - merge_nodes(ids, name) → fusiona N nodos:
          agrega weight, concatena desc/evidence, promedia x/y,
          redirige aristas, elimina duplicados

  4c. commands/graph_events.rs — Eventos
      - clear_ephemeral_nodes(project_id) → DELETE is_ephemeral=1
      - get_recent_graph_events(project_id, source_event?, limit?)

  4d. commands/document/rebuild.rs
      - rebuild_knowledge_graph(project_id) → lanza extractor.py para
        todos los chunks indexados del proyecto

═══ 6. INTEGRACIÓN CON CHAT ═════════════════════════════════════════════════

  6a. ProjectContextPanel.tsx
      - Toggle "Conocimiento territorial" (graph_nodes on/off)
      - Muestra badges de nodos disponibles (label + kind chip)
      - Estado enviado al backend como parte del request

  6b. Inyección en prompt (send_message.rs)
      1. Carga graph_nodes + graph_edges del proyecto desde DB
      2. build_graph_context() filtra por intent del clasificador
      3. Texto inyectado en system prompt del LLM
      4. nodes_used: IDs de nodos seleccionados se guardan en DB
         (JSON array en columna messages.nodes_used)
      5. Ya NO es vec![] — se popula con IDs reales

  6c. Extracción automática post-mensaje (send_message.rs:777-902)
      - Concatena pregunta + respuesta
      - Llama chat_extractor.py vía sidecar --action extract_chat_entities
      - Inserta nodos extraídos con is_ephemeral=true
      - Crea edges entre nodos extraídos
      - Crea nodos web_search desde fuentes de research (is_ephemeral=true)
      - Conecta web_search a nodos extraídos con edge "busqueda web"
      - Emite GraphUpdatePayload vía evento "graph:updated"

  6d. Tipos compartidos (src/types/data.ts)
      - GraphNodeType: 10 kinds
      - BackendGraphNode (raw Rust) → GraphNode (frontend: name→label, kind→type)
      - GraphUpdatePayload: source_event + nodes + edges + timestamp
      - SearchGraphNodesResult: { total, nodes }

═══ 7. FLUJO DE DATOS COMPLETO ═══════════════════════════════════════════════

  Documento:
    Upload → Indexación (chunks) → rebuild_knowledge_graph
      → extractor.py → nodos+edges → DB → GraphPage actualiza

  Chat:
    Mensaje → send_message.rs
      → 1. Carga nodos del grafo → filtra por intent → inyecta en prompt
      → 2. Respuesta LLM → guarda mensaje
      → 3. Extrae entidades del chat (chat_extractor.py)
      → 4. Crea nodos web_search desde fuentes
      → 5. Inserta todo en DB con is_ephemeral
      → 6. Emite "graph:updated" a frontend

  GraphPage:
    Carga inicial → list_graph_nodes + list_graph_edges
    Escucha "graph:updated" → anima nodos nuevos
    Usuario arrastra → update_node_position (persiste)
    Usuario filtra/busca → client-side (kind + substring)
